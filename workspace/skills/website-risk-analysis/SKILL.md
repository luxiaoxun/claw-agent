# Website Risk Analysis Skill

## Overview
本Skill能够使用`web_fetch`工具获取指定URL内容，并结合域名、页面特征、安全上下文等维度，自动完成网站风险分析，输出结构化评估报告。

## When to Use
- 要求分析某个网站是否安全
- 怀疑某个链接是钓鱼网站或恶意网站
- 需要生成一份网站安全分析报告

### 注意事项
- **不要手动输入或猜测页面内容**，必须使用 `web_fetch` 工具获取实际页面数据。
- 如果 `web_fetch` 获取失败，应在报告中明确说明失败原因，并基于 URL 本身进行最小化风险推断。
- 报告应遵循固定模板，语言清晰、结论明确。
- 不要建议用户输入密码或个人信息。

## How to Use

### Step 1: 获取目标 URL 内容
调用 `web_fetch` 工具，传入用户提供的完整 URL

### Step 2: 多维风险分析
基于 URL 特征和获取到的内容（若有），分析以下风险维度：
1.域名可疑性
- 是否为随机字符组合或非常见后缀
- 子域名是否使用 webview、secure、login 等钓鱼常用词
2.URL 参数风险
- 是否包含长随机 token、可疑重定向参数
3.页面内容分析（若获取成功）
- 是否要求输入密码/私密信息
- 是否存在乱码、伪造邮箱、拼写错误
- 是否使用了社会工程话术（如“账号异常”、“保障安全”）
4.技术特征
- 是否使用 HTTPS（不能作为安全保证）
- 是否使用 Cloudflare、CDN 等（中性特征）

### Step 3: 给出结论
根据 Step 2 的分析，综合判定风险等级：
- 高风险：明确钓鱼或恶意特征
- 中风险：可疑但无明确恶意证据
- 低风险：无明显问题
